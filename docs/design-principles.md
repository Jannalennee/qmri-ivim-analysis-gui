# Design Principles for qMRI GUI

## 1) Human-Centered and Role-Aware
Support multiple expertise levels with progressive disclosure.

## 2) Transparency and Trust Calibration
Communicate model capabilities, limits, and uncertainty behavior clearly.

## 3) Interaction and User Control
Allow thresholding, overlays, and parameter tuning instead of fixed outputs.

## 4) Visual Interpretability
Use map-based, familiar imaging metaphors with clear labels and minimal clutter.

## 5) Workflow Integration
Design around real research/clinical task sequences and handoff moments.

## 6) Iterative Evaluation
Continuously test with representative users and feed findings into backlog updates.
